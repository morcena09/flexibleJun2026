Workspace Instructions

Drag the "Use REST APIs" sheet up from the bottom of your screen to choose your testing methodology. You can perform Frontend Record Testing to evaluate real-time UI behavior, or execute REST API Record Tests directly from the developer console to simulate background live operations. Note: Only users authenticated with the Teacher role have permissions to use frontend recording and execute these REST APIs.

  • Module 1: Classroom Layout
    The main page sets up the core viewport and houses the interactive panels. Clicking the floating "i" info button at the bottom right brings up these guidelines at any point during your testing session.
  • Module 2: App Workflow & Lifecycle (app.js)
    When you use the REST tool to change the room state to "Ended" or "Closed", the background application automatically intercepts the event, teardown functions clear the screen, and you are redirected back to the main landing page. If the application detects a recording bot, it automatically hides the helper sheets to keep the recorded video stream clean.
  • Module 3: Live REST Actions (restClient.js)
    The draggable card contains three core integrations which display live API server responses upon execution:
    • Set Classroom State: Change the active room timeline. Select 0 to reset the clock timer, 1 to start the timer, 2 to set the session as ended, or 3 to completely destroy/close the classroom and kick all active users out.
    • Set Recording State: Manage live recording capture streams. Toggle state to 0 to stop recording, or state to 1 to initiate capture. Note: This specialized module dynamically displays the live cURL payload that will be sent to the servers before dispatch.
    • Query Classroom Recording: Used to auditing active streams to ensure there are no rogue recordings left running in the background. It returns and displays the structural Start and End logging timestamps cleanly.
