Workspace Information

Welcome to the Flexible Classroom Workspace! Use this interactive panel to safely generate configurations, sign tokens, and simulate runtime classroom behavior. Follow this sequence to get started:

  • 1. Define the Classroom Setup (Top Form): Fill out the baseline classroom variables like your target infrastructure Region, classroom Layout format, and unique identity keys (Room UUID and User UUID). Please be aware that your generated RTM Tokens and authorized REST API operations are strictly bound to your input Duration.
  • 2. Set Your Identity Profile (Role Type): Choose who you are logging in as. Note that only Teachers (Role Type 1) have permissions to trigger recordings. If you select Recorder (0), Student (2), or Assistant (3), both the user interface recording controls and the backend REST recording utilities will be locked out for you.
  • 3. Input Project Credentials: Scroll down to provide your App ID and App Certificate. These keys are processed locally to sign security handshakes and are never permanently saved.
  • 4. Sign Your Session Tokens: Click the "Generate Security Tokens" button. This populates your read-only token fields. Notice the URL-Safe Encoding Info Bubble: it automatically previews how your raw token characters are sanitized so they don't break web queries.
  • 5. Inspect the Live Code Canvas (Right Panel): As you fill in fields, look over at the simulated code editor window on the right side. It compiles your input fields into a live preview of the initialization payload arguments that the underlying classroom engine consumes.
  • 6. Launch the Runtime Sandbox: Once your configuration looks correct, hit the primary "Launch Environment" button to load your customized runtime container instance.
